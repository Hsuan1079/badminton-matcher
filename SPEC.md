# 羽球配對系統 Spec

## 概覽

一個羽球場館使用的配對管理系統，管理員操作，讓球員公平、有趣地輪流上場打雙打。

---

## 角色

| 角色 | 說明 |
|------|------|
| 管理員 | 新增/刪除球員、觸發配對、結束球局、管理排隊 |
| 球員 | 被動角色，由管理員代為操作 |

> 初始版本無帳號系統，多個管理員共用同一介面（日後接後端時再加認證）。

---

## 核心資料結構

### Player（球員）
```ts
interface Player {
  id: string           // UUID
  name: string         // 顯示名稱
  createdAt: number    // timestamp
}
```

### QueueEntry（排隊中的球員）
```ts
interface QueueEntry {
  id: string
  playerId: string
  joinedAt: number     // 進入排隊的時間，決定等待優先順序
  sessionGamesPlayed: number  // 本次活動已打場次
}
```

### Match（球局）
```ts
interface Match {
  id: string
  team1: [string, string]   // player ids
  team2: [string, string]   // player ids
  courtNumber: number
  startedAt: number
  endedAt: number | null    // null = 進行中
}
```

## 功能需求

### 1. 球員管理
- 新增球員（輸入姓名）
- 刪除球員（軟刪除，保留歷史紀錄）
- 列出所有球員

### 2. 排隊管理
- 球員「報到」：進入排隊池（同一球員不能重複排隊）
- 球員「離場」：從排隊池移除
- 顯示目前排隊人數與順序

### 3. 球局管理
- 系統自動配對（見配對邏輯）
- 管理員手動覆蓋配對（拖拉或選擇）
- 標記球局開始
- 標記球局結束：上場 4 人 sessionGamesPlayed +1，回到排隊池末端（或離場）
- 支援多場地同時進行

### 4. 配對邏輯（核心）

#### 觸發條件
- 管理員手動按下「開始配對」按鈕
- 按鈕在排隊池 < 4 人時 disabled（無法點擊）

#### 選人演算法

**步驟一：建候選池（硬條件）**
1. 找出排隊池中最低場次值 `minGames`
2. 候選池 = 場次 ≤ `minGames + 1` 的所有球員（場次差超過 1 的人不納入）
3. 候選池內按「場次少 → 等最久（joinedAt 小）」排序

**步驟二：選出最佳 4 人組合（新鮮感優先）**

對候選池列舉所有 C(n,4) 組合，每個組合計算**新鮮感總分**：

```
組合分數 = Σ 1 / (1 + 兩人歷史同場次數)   （對 4 人中所有 6 對兩兩組合加總）
```

「同場」定義：**曾在同一場球局出現（不論同隊或對隊）**，搭檔打過與對打都算。

| 兩人同場次數 | 這對的貢獻分數 |
|---|---|
| 0 次（從未同場） | 1.00 |
| 1 次 | 0.50 |
| 2 次 | 0.33 |
| 5 次 | 0.17 |

- 組合分數最高者勝出（最多陌生配對的組合）
- 同分時：選總等待時間最久的組合（joinedAt 加總最小）
- 若候選池剛好 4 人：直接選，跳過列舉

**步驟三：分隊（2v2）**

試所有 3 種分法，用同樣公式對兩組搭檔打分：

```
分隊分數 = 1/(1 + A1與A2同場次數) + 1/(1 + B1與B2同場次數)
```

選分數最高的分法（讓最少同場過的兩人搭檔）。若三種分法分數完全相同，隨機選一種。

#### 例外情況
- 候選池 < 4 人：回傳錯誤，等待更多人排隊
- 所有人都大量同場：演算法仍正常運作，只是分數普遍偏低，結果趨向隨機

## UI 頁面規劃

### 主頁 `/`
- 排隊池列表（顯示姓名、等待時間、本日場次）
- 當前進行中球局（顯示兩隊、開始時間）
- 「開始配對」按鈕（管理員操作）

### 管理員頁 `/admin`
- 球員名單管理（新增/刪除）
- 手動觸發配對
- 重置本次活動（清空場次紀錄，保留球員）
- 歷史球局紀錄

---

## 資料持久化（第一階段）

使用 `localStorage`：
- `bm_players`：球員列表
- `bm_queue`：目前排隊狀態
- `bm_matches`：所有球局紀錄
- `bm_settings`：設定（場地數量等）

---

## 設定

| 設定項 | 預設值 | 說明 |
|--------|--------|------|
| 場地數量 | 2 | 同時可進行的球局數，可自行調整 |
| 場次相似閾值 | 1 | 場次差 ≤ N 視為相似，可納入候選池 |

> 球局結束後 4 人永遠自動回到排隊池尾端（不可關閉）。

---

## 決策紀錄

1. **球局結束後**：4 人自動回到排隊池尾端（不需手動再報到）
2. **場地數量**：可設定，預設 2 面，可自行調整
3. **sessionGamesPlayed 重置**：按日期重置（每天 00:00 自動清零）
4. **手動配對覆蓋**：管理員可無視演算法直接指定上場球員
